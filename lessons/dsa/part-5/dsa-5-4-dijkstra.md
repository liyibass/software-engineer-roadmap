# [dsa-5-4] 最短路徑：Dijkstra 演算法的直覺

> **本章目標**：認識 Dijkstra 演算法——在「加權圖」上找最短路徑的經典方法，建立它的直覺（不需背完整實作），理解它怎麼支撐地圖導航。

## 你會學到

- 加權圖的「最短路徑」問題
- 為什麼 BFS 不夠（要處理權重）
- Dijkstra 的核心直覺：每次選「目前最近的」
- 它的應用（導航）與限制

## 概念說明

### 問題：加權圖的最短路徑

[dsa-5-3] 的 BFS 能在「無權圖」找最短路徑（最少步數）。但真實問題常是**加權圖**（[dsa-5-1]）——邊有「成本」（距離、時間、費用）：

```
地圖導航：從 A 到 B，不是「最少經過幾個路口」，
   而是「總距離/總時間最短」——每段路長度不同（權重）
→ 這時要找的是「權重總和最小」的路徑，BFS 的「最少步數」不夠用。
```

例如「步數少但每段都很長」可能總距離反而比「步數多但每段都短」遠。需要考慮權重的演算法——最經典的是 **Dijkstra（戴克斯特拉）演算法**。

### Dijkstra 的核心直覺：貪婪地選最近的

Dijkstra 的點子很直覺——**從起點開始，每次都「確定一個『目前已知最近』的頂點」，然後用它去更新鄰居的距離，逐步擴展，直到所有點的最短距離都確定**：

```
1. 記錄「從起點到每個頂點的『目前已知最短距離』」
   起點到自己 = 0，到其他 = 無限大（還不知道）
2. 重複：
   從「還沒確定的頂點」中，挑「目前距離最小」的那個 → 確定它
   用它去「鬆弛」鄰居：如果「經過它到鄰居」更短，就更新鄰居的距離
3. 直到所有頂點都確定 → 得到起點到各點的最短距離
```

```mermaid
graph LR
    NOTE["核心：每次貪婪地選『目前最近的未確定點』<br/>確定它的最短距離後，用它更新鄰居<br/>像水波依『實際距離』擴散（近的先確定）"]
```

這張圖點出精髓：Dijkstra 像 BFS 的「加權版」——BFS 一圈圈按「步數」擴散，Dijkstra 按「實際權重距離」擴散，每次確定「目前最近」的點。這種「每步選當下最好」是**貪婪法**的思想（[dsa-6-6] 會深入）。

### 用優先佇列加速

「每次挑『目前距離最小』的頂點」——這正是**優先佇列（[dsa-4-5] 的堆積）** 的拿手好戲！用優先佇列（最小堆積）來「總是快速取出距離最小的點」，能讓 Dijkstra 高效運作。這是堆積/優先佇列的經典應用，把前面學的串起來了。

```
所以 Dijkstra 動用了好幾個前面的知識：
   圖（dsa-5-1）+ 鄰接串列（dsa-5-2）+ 優先佇列/堆積（dsa-4-5）+ 貪婪思想（dsa-6-6）
→ 這就是資料結構與演算法「環環相扣」的展現。
```

### 應用與限制

```
應用：
   地圖導航（Google Maps 等找最短路線的核心思想）
   網路路由（找延遲最低的路徑，cs 課程 Part 6-3）
   任何「加權圖找最短/最便宜路徑」的問題

限制：
   Dijkstra 要求「邊的權重不能是負的」
   （有負權重要用別的演算法如 Bellman-Ford）
   原因：它的貪婪假設「已確定的點不會再變短」，負權重會打破這假設
```

**你不需要背 Dijkstra 的完整程式碼**——重點是理解它的直覺（貪婪地一步步確定最近點、用優先佇列加速）和它解決的問題（加權最短路徑）。真正要用時，多數場景有現成函式庫。

## 範例：導航的本質

```
你用導航 App 從家到公司：
   地圖 = 加權圖（路口=頂點，道路=邊，車程=權重）
   App 跑類似 Dijkstra 的演算法：
      從你家開始，一步步確定「到各路口的最短車程」
      最終得出「到公司的最短路線」
   （真實導航還會用更進階的優化，但核心思想源於 Dijkstra）

→ 你每天用的導航，背後是圖論最經典的演算法在運作。
```
