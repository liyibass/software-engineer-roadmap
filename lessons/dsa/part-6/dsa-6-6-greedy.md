# [dsa-6-6] 貪婪法（Greedy）：每步都選當下最好，何時對、何時錯

> **本章目標**：認識貪婪法——「每一步都選當下看起來最好的」這種簡單直覺的策略，理解它何時有效、何時會出錯。

## 你會學到

- 貪婪法的核心：每步選局部最優
- 它的優點：簡單、快
- 它的陷阱：局部最優 ≠ 全域最優
- 怎麼判斷貪婪法可不可行

## 概念說明

### 貪婪：每步選當下最好

**貪婪法（Greedy）** 是一種解題策略——**在每一步，都選「當下看起來最好的選項」，不回頭、不考慮長遠**，希望這樣一路下來能得到整體最好的答案。

比喻：

```
爬山想到最高峰，貪婪策略 = 「每一步都往『當下最陡上坡』走」。
   單純、直接：不規劃路線，只看眼前哪邊最高就往哪走。
```

最經典的例子是**找零錢**：

```
問題：用最少的硬幣湊出 36 元（有 1, 5, 10, 50 元硬幣）
貪婪策略：每次都拿「不超過剩餘金額的最大硬幣」
   36 → 拿 10（剩 26）→ 拿 10（剩 16）→ 拿 10（剩 6）→ 拿 5（剩 1）→ 拿 1
   = 用了 6 個硬幣（10×3 + 5 + 1）
→ 對台幣這種幣值，貪婪剛好給出最優解！
```

### 優點：簡單又快

貪婪法的魅力在於——**它非常簡單、非常快**。不用考慮所有可能（那可能是指數級 O(2ⁿ)，[dsa-1-2]），只要「每步挑當下最好」，通常是 O(n) 或 O(n log n)。當貪婪「正確」時，它是最優雅高效的解法。

你已經見過貪婪的應用——[dsa-5-4] 的 **Dijkstra 演算法**就是貪婪（每次選「目前最近的點」）。

### 陷阱：局部最優 ≠ 全域最優

但貪婪有個**致命陷阱**——**「每步選當下最好」不一定導致「整體最好」**！局部最優的累積，可能錯過真正的最優解：

```mermaid
graph TB
    NOTE["貪婪『只看眼前』<br/>可能走進『眼前最好、但整體更差』的死路<br/>→ 局部最優 ≠ 全域最優"]
```

回到爬山比喻——「每步往最陡上坡走」可能把你帶到一個**小山丘的頂**（局部最高），卻錯過遠處更高的**真正高峰**，因為去那座高峰中間要先「下坡」（而貪婪不肯下坡）。

一個找零錢的反例：

```
假設幣值是 1, 3, 4 元，要湊 6 元：
   貪婪：先拿最大的 4（剩 2）→ 拿 1（剩 1）→ 拿 1 = 3 個硬幣（4+1+1）
   最優：其實拿 3+3 = 只要 2 個硬幣！
→ 貪婪在這組幣值下「出錯了」——它被「先拿最大的 4」這個局部最優騙了。
```

### 怎麼判斷貪婪可不可行

```
貪婪法「不保證」總是對——能不能用，要看問題的性質：
   有些問題「貪婪剛好正確」（如台幣找零、Dijkstra、霍夫曼編碼）
      → 這類問題有「貪婪選擇性質」，能證明每步局部最優導致全域最優
   有些問題「貪婪會錯」（如上面 1,3,4 找零、背包問題）
      → 要用別的方法，例如動態規劃（dsa-6-7，它會考慮更全面）

→ 實務建議：想到貪婪解時，要「驗證它真的對」（找反例、或證明），
  別假設「每步最好就一定整體最好」。這是貪婪最大的坑。
```

這呼應 [dsa-0-3] 的「正確第一」——一個又快又簡單但**會算錯**的貪婪解，毫無價值。貪婪很誘人，但要先確認它對。

## 程式碼範例

```typescript
// 找零錢的貪婪法（對「台幣這種幣值」正確）
function coinChangeGreedy(amount: number, coins: number[]): number {
  // coins 要由大到小排序
  const sorted = [...coins].sort((a, b) => b - a);
  let count = 0;
  let remaining = amount;
  for (const coin of sorted) {
    while (remaining >= coin) {     // 盡量多拿當下最大的硬幣
      remaining -= coin;
      count++;
    }
  }
  return count;
}

console.log(coinChangeGreedy(36, [1, 5, 10, 50]));   // 6（貪婪正確）
console.log(coinChangeGreedy(6, [1, 3, 4]));         // 3（貪婪錯！最優是 2）
```

說明：同一個貪婪演算法，對 `[1,5,10,50]` 給出最優解，對 `[1,3,4]` 卻不是最優——**完美示範「貪婪不一定對」**。這提醒你：用貪婪前，務必確認它對你的問題真的正確（或接受它只是「夠好的近似解」）。

## 小練習

1. 用「爬山」的比喻，解釋貪婪法為什麼可能「卡在小山丘、錯過真正高峰」。
2. 用幣值 `[1, 3, 4]` 湊 6 元，手動跑一遍貪婪，看它為什麼給出 3 個（而非最優的 2 個）。
