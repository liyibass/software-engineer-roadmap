### S3 是什麼（先知道這些就好）

**S3（Simple Storage Service，簡單儲存服務）** 是 AWS 最老牌、最常用的服務之一。一句話：

> **S3 是一個「放檔案的雲端空間」——你把檔案丟上去，它幫你穩穩存著，還能讓全世界透過網址存取。**

兩個關鍵詞：

- **Bucket（儲存桶）**：你的「資料夾 / 容器」，檔案都放在 bucket 裡。bucket 的名字**全球唯一**（全 AWS 不能重複）。
- **Object（物件）**：你放進去的每個檔案，叫一個 object。

S3 還有個方便功能——**靜態網站託管（static website hosting）**：你把網頁的 HTML 檔丟進 bucket、開啟這個功能，S3 就會給你一個網址，全世界都能連到你的網頁。**不用伺服器、不用設定，就能上線一個網站。**

## 程式碼範例

> 以下在 AWS Console 操作。記得先確認**右上角的 Region**（aws-1-2 學的）選好了，例如東京。

### 第一步：準備一個網頁

在你電腦建一個簡單的 `index.html`：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>我的第一個 AWS 網站</title>
</head>
<body>
  <h1>🎉 這個網頁跑在 AWS S3 上！</h1>
  <p>我成功把網站公開到全世界了。</p>
</body>
</html>
```

### 第二步：建立一個 S3 Bucket

1. Console 搜尋進入 **S3**。
2. **Create bucket**。
3. Bucket 名稱：取一個**全球唯一**的名字（例如 `my-first-site-你的名字-1234`）。
4. **取消勾選「Block all public access」**（因為我們要讓它公開）——它會跳警告要你確認，因為這是「公開」設定。
   > ⚠️ 一般情況下，這個「封鎖公開存取」是該開著的安全防線。這裡因為是「刻意要做公開網站」才關掉。Part 5 會深入這個安全考量。
5. 其他用預設，建立。

### 第三步：上傳網頁

1. 進入剛建的 bucket → **Upload** → 把 `index.html` 拖進去 → Upload。

### 第四步：開啟靜態網站託管

1. 進入 bucket 的 **Properties（屬性）** 分頁。
2. 滑到最下面 **Static website hosting** → Edit → Enable。
3. Index document 填 `index.html`。
4. 儲存。它會給你一個**網站端點網址**（像 `http://my-first-site-....s3-website-....amazonaws.com`），記下來。

### 第五步：設定 Bucket Policy（允許公開讀取）

光開啟託管還不夠，要再給一個「政策」允許大家讀取檔案。

1. 進入 bucket 的 **Permissions（權限）** 分頁 → **Bucket policy** → Edit。
2. 貼上這段（把 `你的bucket名稱` 換掉）：

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::你的bucket名稱/*"
    }
  ]
}
```

這段 JSON 是一份「政策」——它說「**允許（Allow）任何人（Principal: \*）讀取（s3:GetObject）這個 bucket 裡的所有物件**」。你現在不用懂每個欄位，Part 2-3 會逐行解析這種 IAM/政策語法。儲存。

### 第六步：見證成就 🎉

打開第四步記下的那個網站網址。你應該看到你的網頁——**「🎉 這個網頁跑在 AWS S3 上！」**

恭喜！你剛剛**把一個網站公開到了全世界**，任何人用這個網址都連得到。而且這幾乎是免費的（在 Free Tier 內）。這就是你的第一個 AWS 成就。

### ⚠️ 別忘了：學完可以清理

如果這只是練習、之後不需要了，記得依 aws-1-3 的「關資源習慣」清理：把 bucket 裡的檔案刪掉、再刪掉 bucket。S3 存放在 Free Tier 內幾乎不花錢，但養成「實驗完清理」的好習慣很重要。

### 練習 3：思考限制

這個「S3 靜態網站」能放 HTML、圖片、CSS、JS。但它能跑「後端程式」（例如處理登入、連資料庫）嗎？為什麼不行？

> 提示：S3 只是「放檔案、回傳檔案」，它不會「執行程式」。要跑後端，需要會運算的東西——那就是 Part 3 的 EC2、Part 8 的 Lambda。這也預告了你接下來的學習。
