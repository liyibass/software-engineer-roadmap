# [aws-4-2] CIDR：10.0.0.0/16 這串數字代表多少 IP

> **本章目標**：看懂 CIDR 表示法，知道 `/16`、`/24` 這些數字代表多少 IP 位址，能為 VPC 和子網路規劃合理的 IP 範圍。

## 你會學到

- CIDR 表示法是什麼、怎麼讀
- `/16`、`/24` 後面的數字代表什麼
- 怎麼為 VPC 和子網路分配 IP 範圍
- 規劃 IP 範圍的實用原則

## 概念說明

### 為什麼要懂 CIDR

建 VPC 的第一件事，就是給它一個 **IP 位址範圍**，格式長這樣：`10.0.0.0/16`。子網路也要分到範圍，如 `10.0.1.0/24`。這個 `/16`、`/24` 的寫法叫 **CIDR**。

看不懂 CIDR，你就無法規劃 VPC（不知道該給多大、子網路怎麼切）。好消息是——你 infra Part 3-1 已經碰過 IP 和這個概念了，這章把它講清楚。其實搞懂一個重點就會了。

---

### 先複習：IP 位址

一個 IPv4 位址長這樣：`10.0.1.25`——四組數字，每組 0~255，用點分隔。它是一台機器在網路上的「門牌號碼」（infra Part 3-1）。

每組數字（0~255）其實是 **8 個位元（bit）**，所以一個完整 IP 是 **32 個位元**。記住「**總共 32 位元**」，是理解 CIDR 的關鍵。

---

### CIDR：用「/數字」表示一個範圍

CIDR 表示法是 `IP位址/數字`，例如 `10.0.0.0/16`。那個斜線後的數字，意思是：

> **「前面幾個位元是『固定的網路部分』，剩下的位元可以自由變化（給不同機器用）」。**

```
10.0.0.0/16
         │
         └── 前 16 個位元固定，剩下 32-16 = 16 個位元可變
             → 可變的位元越多，能容納的 IP 越多
```

關鍵公式：**可用 IP 數量 ≈ 2^(32 − 斜線數字)**

- `/16` → 2^(32-16) = 2^16 = **約 65,536 個** IP（很大的範圍）
- `/24` → 2^(32-24) = 2^8 = **256 個** IP（中等範圍）
- `/28` → 2^(32-28) = 2^4 = **16 個** IP（小範圍）

**口訣：斜線後的數字越小，範圍越大；越大，範圍越小。** （因為數字越小，「可變的位元」越多。）

---

### 視覺化：範圍大小

```mermaid

    B["/24<br/>256 個 IP<br/>（中範圍，給一個子網路）"]

```

實務上的慣例：

- **VPC 用 `/16`**（約 6.5 萬個 IP）——給整個社區一個夠大的範圍。
- **子網路用 `/24`**（256 個 IP）——從 VPC 的大範圍裡，切出一塊塊給各子網路。

---

### 怎麼讀一個範圍涵蓋哪些 IP

以 `10.0.1.0/24` 為例：

- `/24` 代表「前 24 位元固定」= 前三組數字 `10.0.1` 固定。
- 最後一組（第 4 組）可變（0~255）。
- 所以這個範圍涵蓋 `10.0.1.0` ~ `10.0.1.255`，共 256 個 IP。

再看 `10.0.0.0/16`：

- `/16` = 前兩組 `10.0` 固定。
- 後兩組可變。
- 涵蓋 `10.0.0.0` ~ `10.0.255.255`，約 6.5 萬個 IP。

> 你不用會手算二進位——記住「**`/24` 看前三組、最後一組可變（256 個）；`/16` 看前兩組、後兩組可變（6.5 萬個）**」這兩個最常用的，就能應付九成情況。需要精算時用線上 CIDR 計算器。

---

### 為什麼用 10.0.x.x 這種位址

你會注意到 VPC 常用 `10.0.0.0/16`、`172.16.x.x`、`192.168.x.x` 這些位址。這些是**「私有 IP 範圍」**（infra Part 3-1 學過）——專門保留給「內部網路」用、不會和公開網際網路衝突的位址。VPC 是你的私有網路，所以用私有 IP 範圍。`10.0.0.0/16` 是最常見的選擇。

---

### 規劃的實用原則

**① VPC 給夠大的範圍**：用 `/16`，留足成長空間。IP 範圍一旦設定，要改很麻煩，所以一開始就給大一點。

**② 子網路規律地切**：用 `/24`，並規律命名，例如：
- `10.0.1.0/24` → AZ-a 的公開子網路
- `10.0.2.0/24` → AZ-a 的私有子網路
- `10.0.3.0/24` → AZ-b 的公開子網路
- `10.0.4.0/24` → AZ-b 的私有子網路

**③ 子網路範圍不能重疊**：同一個 VPC 裡，各子網路的 IP 範圍不能互相重疊（就像社區裡不能有兩戶同地址）。

**④ 預留成長空間**：別把 VPC 的範圍一次切光，留一些給未來新增的子網路。

## 範例：規劃一個 VPC 的 IP

```
規劃一個 VPC，要支援「跨 2 個 AZ、各有公開+私有子網路」：

VPC：10.0.0.0/16
  → 約 65,536 個 IP，夠大，留足成長空間

切成 4 個子網路（各 /24，256 個 IP）：
  10.0.1.0/24  → AZ-a 公開子網路（放負載平衡器、對外伺服器）
  10.0.2.0/24  → AZ-a 私有子網路（放應用、資料庫）
  10.0.3.0/24  → AZ-b 公開子網路（高可用：跨 AZ）
  10.0.4.0/24  → AZ-b 私有子網路

  → 範圍都不重疊，命名規律，還剩 10.0.5.0 之後一大片給未來擴充

這就是 aws-4-8 你要親手建的 VPC 的 IP 規劃藍圖。
```

注意這個規劃怎麼體現原則：VPC 用大範圍（/16）、子網路規律切（/24）、不重疊、跨 AZ（為了 4-7 的高可用）、留成長空間。這就是專業的 VPC IP 規劃。
