# [cs-2-2] 布林代數：用數學描述邏輯

> **本章目標**：認識布林代數——一套「用代數方式操作 true/false」的數學，它是邏輯閘的理論語言，也是你寫程式時邏輯運算的基礎。

## 你會學到

- 布林代數是什麼、和邏輯閘的關係
- 基本運算：AND（·）、OR（+）、NOT（′）
- 幾個實用的化簡定律
- 為什麼化簡邏輯式很重要

## 概念說明

### 邏輯也能「算」

[cs-2-1] 的邏輯閘是「硬體」。但在設計它們之前，工程師需要一套「**用紙筆推演邏輯**」的數學工具——這就是 **布林代數（Boolean algebra）**，由數學家 George Boole 創立。

布林代數的世界裡，變數只有兩個值：**真（1）和假（0）**。它定義了怎麼「運算」這些真假值，就像普通代數運算數字一樣。對應 [cs-2-1] 的三個閘：

```
AND（且）寫成 ·（像乘法）   A · B
OR （或）寫成 +（像加法）   A + B
NOT（非）寫成 ′ 或 上面一橫  A′
```

### 為什麼像乘法和加法？

這個符號選得很巧妙——布林運算的行為，真的很像乘法和加法：

```
AND 像乘法：
   1 · 1 = 1    1 · 0 = 0    0 · 0 = 0
   （只有「全是 1」才得 1，就像乘法只要有一個 0 結果就是 0）

OR 像加法：
   0 + 0 = 0    0 + 1 = 1    1 + 1 = 1（注意：不是 2，布林只有 0/1）
   （只要有一個 1 結果就是 1）
```

所以你可以「像算數學一樣」操作邏輯式。這讓設計複雜電路變成「可以推演、可以化簡」的數學問題。

### 實用的化簡定律

布林代數有一些定律，能把複雜的邏輯式**化簡**。幾個直覺的：

```
A · 1 = A          （和 1 做 AND，等於原本）
A · 0 = 0          （和 0 做 AND，一定是 0）
A + 0 = A          （和 0 做 OR，等於原本）
A + 1 = 1          （和 1 做 OR，一定是 1）
A · A = A          （自己和自己 AND，還是自己）
A + A′ = 1         （一個東西「或」它的相反，一定成立）
```

還有一個很重要、很實用的 **狄摩根定律（De Morgan's Law）**：

```
(A · B)′ = A′ + B′      「不是(A且B)」= 「非A 或 非B」
(A + B)′ = A′ · B′      「不是(A或B)」= 「非A 且 非B」
```

狄摩根定律告訴你「**否定怎麼穿過 AND/OR**」——把 NOT 往裡面推時，AND 要變 OR、OR 要變 AND。這在化簡邏輯、寫程式條件時超常用。

### 為什麼要化簡？

化簡邏輯式有實際的價值：

```mermaid
graph LR
    COMPLEX["複雜的邏輯式<br/>要很多邏輯閘"]
    SIMPLE["化簡後的邏輯式<br/>需要的閘變少"]
    COMPLEX -->|"布林代數化簡"| SIMPLE
    SIMPLE --> BENEFIT["更少零件 → 更便宜<br/>更省電、更快"]
```

這張圖在說：**邏輯式越簡單，需要的邏輯閘越少**——電路就更便宜、更省電、更快。在晶片裡，每省下一個閘乘以幾十億個電路，效益巨大。所以布林代數的化簡，不只是數學練習，而是省下真金白銀的工程。

## 範例：用狄摩根定律改寫程式條件

布林代數在你寫程式時就用得到。看這個常見的化簡：

```
原本的條件（有點繞）：
   if ( !(已登入 && 已驗證) ) { 要求登入 }

用狄摩根定律 (A·B)′ = A′ + B′ 改寫：
   if ( !已登入 || !已驗證 ) { 要求登入 }

→ 「不是(已登入且已驗證)」等於「沒登入 或 沒驗證」，
   後者通常更好讀。這就是布林代數在日常的應用。
```

## 小練習

1. 用布林代數定律化簡：`A · 1 + A · 0`（提示：先算各項，再 OR）。
2. 用狄摩根定律改寫：「不是（是假日 或 請假）」會等於什麼？
3. 思考題：為什麼把邏輯式化簡，能讓晶片更省電、更便宜？（提示：和需要幾個邏輯閘有關。）
