## 概念說明

### 為什麼不直接用檔案？

你可能想：「要存資料，用檔案（[cs-5-6]）存不就好了？」對小東西可以，但當資料變多、變複雜、要多人同時用，純檔案就不夠了：

```
用純檔案存大量資料的麻煩：
   查找慢：要找「所有 30 歲的使用者」，得自己讀整個檔案逐筆比對
   難維護關聯：使用者和他的訂單怎麼關聯？自己管很容易亂
   並行危險：多人同時改同一個檔案 → 競爭條件（cs-5-5）、資料壞掉
   沒保障：寫到一半當機 → 資料可能殘缺
```

**資料庫（database）** 就是專門解決「**高效、可靠、並行地管理大量結構化資料**」的系統。它幫你處理掉上面所有麻煩。

```
比喻：純檔案像「把資料隨手記在一堆筆記本」。
     資料庫像「一套專業的檔案管理系統」——
     有索引能快速查、有規則保證一致、能多人安全共用。
```

### 資料庫管理系統幫你做的事

資料庫系統（DBMS）提供幾個關鍵能力：

```
高效查詢：用「索引」（像書的目錄）讓查找超快，不用翻整本
   → 呼應 dsa 的資料結構、[課外讀物 E-4](../../../課外讀物/E-4-database/E-4-1-what-is-index.md)
一致性保障：用「交易（transaction）」確保「要嘛全做完，要嘛全不做」
   → 例如轉帳：扣款和入帳必須同時成功，不能只做一半
並行控制：多人同時存取也不會弄壞資料（處理 cs-5-5 的競爭）
持久可靠：資料安全保存，當機也有保護機制
```

### 關聯式模型：用表格組織

最主流的資料庫是**關聯式資料庫（relational database）**，它用一個很直覺的方式組織資料——**表格（table）**，就像試算表：

```
使用者表 (users)
┌─────┬────────┬─────┐
│ id  │ name   │ age │
├─────┼────────┼─────┤
│ 1   │ 小美   │ 28  │
│ 2   │ 阿明   │ 35  │
└─────┴────────┴─────┘
每一「列(row)」是一筆資料，每一「欄(column)」是一個欄位。
```

這個「表格」對應你在 **rust 課程 [rust-9-4]** 用 sqlx 操作的 PostgreSQL 表、[rust-3-1] 的 struct（一列 ≈ 一個 struct 實例）。

### 「關聯」：表格之間的連結

「關聯式」的精髓在——**不同表格之間可以「關聯」起來**。例如「訂單表」用一個欄位指向「使用者表」，表示「這筆訂單屬於哪個使用者」：

```mermaid
graph LR
    U["使用者表<br/>id=1 小美"]
    O["訂單表<br/>order#100, user_id=1"]
    O -->|"user_id 指向 users.id"| U
```

這張圖在說：訂單表用 `user_id` 欄位「關聯」到使用者表——這樣就能表達「小美下了訂單 100」，而不用把使用者資料重複抄在每筆訂單裡。這種「用關聯避免重複、保持一致」是關聯式模型的核心優點。

### SQL：和資料庫對話的語言

操作關聯式資料庫用一種專門的語言——**SQL（Structured Query Language）**。它讓你用接近英文的方式下指令：

```sql
SELECT name FROM users WHERE age > 30;
-- 「從 users 表，選出 age 大於 30 的人的 name」
```

你在 **rust 課程 [rust-9-4]** 寫的 `SELECT ... FROM todos` 就是 SQL。SQL 是後端工程師的必備技能。

### 也有「非關聯式」資料庫

順帶一提，除了關聯式（SQL）資料庫，還有**非關聯式（NoSQL）** 資料庫——它們不用嚴格的表格，而用其他模型（文件、鍵值對、圖…），在某些場景（超大規模、彈性結構）更合適。你之後可能聽到 MongoDB、Redis（**快取課程** Part 5）等。各有適用場景。

## 範例：為什麼用資料庫

```
一個購物網站要管理「使用者」和「訂單」：
   用純檔案：查「小美的所有訂單」要讀整個訂單檔逐筆比對；
            多人同時下單可能弄亂檔案；當機可能資料殘缺。
   用資料庫：users 表 + orders 表（用 user_id 關聯）；
            用 SQL 一句 `SELECT * FROM orders WHERE user_id = 1` 快速查；
            交易保證「下單扣庫存」一致；多人安全並行。

→ 這就是為什麼幾乎所有正式系統都用資料庫存資料。
```
