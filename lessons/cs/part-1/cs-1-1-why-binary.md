# [cs-1-1] 為什麼是二進位？電腦只懂「開」與「關」

> **本章目標**：理解為什麼電腦用「二進位」（只有 0 和 1）來表示一切，這個選擇背後的物理原因，以及 0 和 1 怎麼成為數位世界的磚塊。

## 你會學到

- 「二進位」是什麼意思
- 為什麼電腦不用我們熟悉的十進位
- 0 和 1 對應到電子元件的什麼狀態
- 「bit（位元）」這個最小單位

## 概念說明

### 電腦的世界只有兩種狀態

我們人類用**十進位**（0~9 共十個數字）算數，大概是因為我們有十根手指。但電腦不用十進位，它用**二進位**——**只有 0 和 1 兩個數字**。

為什麼？答案藏在「電腦是電子機器」這件事裡。電腦內部的元件，最容易、最可靠地表達的就是**兩種狀態**：

```
有電 / 沒電
開   / 關
高電壓 / 低電壓
```

把這兩種狀態，一個叫 **1**、一個叫 **0**，二進位就誕生了。比喻一下：

```
電腦像一整面「電燈牆」，每盞燈只有「亮」或「不亮」。
亮 = 1，不亮 = 0。
用很多盞燈的「亮暗組合」，就能表達任何資訊。
```

### 為什麼不用十進位？因為「兩種狀態」最可靠

你可能想：「那做出能表達十種狀態的元件不就好了？」技術上可以，但**不可靠**。

想像要用「電壓高低」表達十個數字——你得把電壓切成十個精確的等級（0.0V、0.5V、1.0V…）。但電子訊號會有雜訊、會飄移，10 個等級很容易「認錯」（0.5V 到底算第幾級？）。

而「只分兩種」就穩很多——**只要分辨「有沒有電」**，中間留很大的容錯空間，幾乎不會認錯。

```mermaid
graph TB
    A["十進位元件<br/>要分 10 個電壓等級<br/>→ 容易因雜訊認錯 ❌"]
    B["二進位元件<br/>只分『有電/沒電』<br/>→ 容錯大、超可靠 ✅"]

```

這張圖在說：用「兩種狀態」雖然每個位元能裝的資訊少，但**極度可靠**——而可靠正是電腦最需要的。寧可用很多個「簡單可靠」的 0/1，也不要少數「複雜易錯」的多狀態元件。這就是二進位勝出的根本原因。

### bit：最小的資訊單位

一個「只能是 0 或 1」的位置，就是資訊的最小單位——**位元（bit，binary digit 的縮寫）**。

```
1 個 bit = 一盞燈 = 能表達 2 種可能（0 或 1）
```

一個 bit 能表達的東西很有限（就兩種）。但只要**把很多 bit 排在一起**，能表達的組合就爆炸性成長：

```
1 個 bit：  2 種（0、1）
2 個 bit：  4 種（00、01、10、11）
3 個 bit：  8 種（000 ... 111）
n 個 bit：  2 的 n 次方 種
```

每多一個 bit，可能的組合就翻倍。所以用足夠多的 bit，就能表達數字、文字、圖片、聲音——**萬物皆可化為 0 和 1**。這是 Part 1 接下來幾章的主題：各種東西「怎麼被編碼成一串 0 和 1」。

## 範例：用 bit 數一數能表達幾種

感受一下「翻倍」的威力：

```
要表達「一週七天」，需要幾個 bit？
   3 個 bit → 8 種組合（夠表達 7 天，還多一種）
   2 個 bit → 只有 4 種（不夠）
→ 所以 3 個 bit 剛好夠。

要表達「26 個英文字母」，需要幾個 bit？
   4 個 bit → 16 種（不夠）
   5 個 bit → 32 種（夠了）
→ 所以至少 5 個 bit。
```

這種「要表達 N 種東西，需要幾個 bit」的計算，是理解資料儲存的基本功。

## 小練習

1. 用自己的話解釋：為什麼電腦用二進位而不是十進位？關鍵的「可靠」差別在哪？
2. 算算看：4 個 bit 能表達幾種不同的組合？8 個 bit 呢？（提示：2 的幾次方）
3. 思考題：如果要表達「一年 12 個月」，至少需要幾個 bit？（提示：找出「2 的幾次方」第一次超過 12。）
