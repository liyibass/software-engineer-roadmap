# [cs-6-3] IP、封包與路由：資料怎麼找到對方（簡介）

> **本章目標**：理解資料在網路上怎麼「找到正確的目的地」——IP 位址像門牌、路由像導航，以及 TCP 怎麼確保資料可靠送達。

## 你會學到

- IP 位址：每台裝置的「網路門牌」
- 路由（routing）：封包怎麼一站站找到目的地
- TCP vs UDP：可靠 vs 快速
- 為什麼這套設計這麼強韌

## 概念說明

> 這是「自成一體的簡介」。網路的完整深入內容在 [課外讀物 E-3](../../../課外讀物/E-3-network/E-3-1-how-internet-works.md)，這裡建立核心直覺。

### IP 位址：網路上的門牌

[cs-6-1] 說網路像郵政系統。要寄信，得有「地址」——在網路上，這個地址叫 **IP 位址（IP address）**。每台連網的裝置都有一個 IP，用來在網路上被找到：

```
IPv4 位址長這樣：192.168.1.10（四組數字）
IPv6 位址更長（因為 IPv4 不夠用了）

你的手機、筆電、每台伺服器，在網路上都有 IP 位址，
就像每棟房子都有門牌號碼。
```

封包（[cs-6-1]）上會寫著「來源 IP」和「目的 IP」——就像信封上的寄件人和收件人地址。網路靠這個把封包送到對的裝置。

### 路由：一站一站找過去

但網際網路這麼大，封包怎麼從你家「找到」地球另一端的伺服器？靠**路由（routing）**——封包不是「直達」，而是**經過一個個「路由器（router）」接力轉送**，每個路由器決定「下一站往哪送」：

```mermaid
graph LR
    YOU["你的裝置"] --> R1["路由器1<br/>(你家)"]
    R1 --> R2["路由器2<br/>(電信商)"]
    R2 --> R3["路由器3<br/>...更多..."]
    R3 --> DEST["目的伺服器"]
```

這張圖在說：封包像「轉好幾次車的旅程」——每個路由器看封包的目的 IP，決定「往哪個方向轉送一步」，一站接一站，最終抵達目的地。比喻：

```
路由像「問路接力」：
   你不知道完整路線，但每到一個路口，
   那裡的指路人（路由器）告訴你「往這個方向走下一步」。
   一站站問下去，最終就到了。
```

神奇的是——**同一份資料的不同封包，可能走不同路徑**（哪條順走哪條），最後在目的地重組。這讓網路非常**強韌**：某條線路斷了，封包自動繞道。這個「去中心化、能繞道」的設計，是網際網路當年為了「即使部分節點被摧毀也能運作」而生的。

### TCP vs UDP：可靠 vs 快速

封包會到目的地，但「**會不會掉、順序對不對**」要誰管？這是傳輸層（[cs-6-2]）TCP/UDP 的工作，兩者取捨不同：

| | TCP | UDP |
|---|-----|-----|
| 可靠性 | **高**：確保送達、順序正確、掉了重傳 | 低：盡力送，掉了就掉了 |
| 速度 | 較慢（要確認、重傳的開銷）| **快**（不囉嗦）|
| 適合 | 網頁、檔案、訊息（不能錯）| 直播、線上遊戲、語音（要即時，掉一兩格沒差）|

```
TCP 像「掛號信」：確認對方收到、會補寄遺失的——可靠但慢一點。
UDP 像「明信片」：寄出去就算了，不保證到——快，但可能掉。

你看網頁、下載檔案 → TCP（內容不能錯一個字）
你看直播、打即時遊戲 → 常用 UDP（要即時，偶爾掉格比卡頓好）
```

### 還有一個幫手：DNS

你上網時打的是 `google.com`，不是 IP 數字。但網路是靠 IP 找路的——中間靠 **DNS（網域名稱系統）** 幫你「把好記的網址翻譯成 IP」，像網路的電話簿。

```
你打 google.com → DNS 查出它的 IP → 才能用 IP 去找它
```

## 範例：一個封包的旅程

```
你的手機要連 example.com：
   1. DNS：把 example.com 翻成 IP（例如 93.184.216.34）
   2. 封包寫上「目的 IP = 93.184.216.34」
   3. 路由：封包經過你家路由器 → 電信商 → 多個路由器接力
      每站看目的 IP，決定下一步往哪轉
   4. 抵達 example.com 的伺服器
   5. TCP：確保所有封包都到齊、順序正確、沒掉的就重傳
   6. 組合成完整資料 → 你看到網頁

→ IP 負責「定址與找路」，TCP 負責「可靠送達」，DNS 負責「翻譯網址」。
  三者合作，你才能順利上網。
```
