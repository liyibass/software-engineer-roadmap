# [E-11-2] 瀏覽器快取：Cache-Control 標頭怎麼設（簡介）

## 瀏覽器快取：重複造訪超快的關鍵

你第二次打開同一個網站，通常比第一次快很多——因為**瀏覽器把上次下載的資源（圖片、CSS、JS）存在你電腦裡了**，第二次直接從本機拿，不用重新下載。這就是**瀏覽器快取**，是前端效能的重要一環（E-11-1）。

它帶來：使用者更快、省流量、減輕伺服器負擔。

## 關鍵：用 Cache-Control 標頭控制

瀏覽器要不要快取、快取多久，是**伺服器透過 HTTP 回應的 `Cache-Control` 標頭**告訴瀏覽器的：

```
Cache-Control: max-age=3600
```

意思是「這個資源可以快取 3600 秒（1 小時）」。1 小時內瀏覽器直接用快取、不發請求（最快）。

幾個常見指令（簡述，完整版見快取課程 cache-3-2）：

| 指令 | 意思 |
|------|------|
| `max-age=N` | 可快取 N 秒 |
| `no-store` | 完全不快取（敏感資料）|
| `no-cache` | 可存，但用前要先驗證（不是「不快取」！）|
| `public` / `private` | 能否被 CDN 等共享快取 |

## 一個必知的坑：前端更新後使用者看到舊版

瀏覽器快取最有名的坑——**你更新了前端，使用者卻還看到舊版**（因為他的瀏覽器還用著快取的舊檔）。

簡短的解法：**靜態檔用「檔名加 hash」（內容變檔名就變，瀏覽器自然下載新的）+ 設永久快取；index.html 設 no-cache（總是拿最新的指路牌）。**

> 這個坑的完整解析、雙層（瀏覽器+CDN）的處理、Cache-Control 各指令的細節——都在**快取課程 cache-3-2、cache-3-5、cache-4-4**。這篇只是讓你在「效能」的脈絡下知道「有瀏覽器快取這回事、靠 Cache-Control 控制」。

## 小結

- 瀏覽器快取讓「重複造訪」超快（資源存本機，不重新下載）。
- 靠伺服器的 **`Cache-Control`** 標頭控制（max-age / no-store / no-cache / public-private）。
- 經典坑：前端更新後看到舊版 → 用 hash 檔名 + index.html no-cache 解決。
