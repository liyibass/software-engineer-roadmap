# [E-5-4] Hello World 的歷史

> **目標**：輕鬆認識「Hello, World!」這個全世界程式設計師的「第一個程式」傳統，是怎麼來的。

## 全世界的「第一個程式」

幾乎每個學程式的人，寫的第一個程式都是讓電腦印出：

```
Hello, World!
```

不管什麼語言——Python、JavaScript、C、Java——教學的第一課幾乎都是「印出 Hello World」。這成了全球程式設計師的「**入門儀式**」。但這個傳統是怎麼來的？

## 來自一本經典教科書

「Hello, World」的流行，主要來自一本 1978 年的經典著作：

> **《The C Programming Language》**（C 程式語言），作者是 **Brian Kernighan** 和 **Dennis Ritchie**（Ritchie 是 C 語言和 Unix 的發明者之一）。這本書常被簡稱為「**K&R**」。

書的第一個範例程式，就是印出「hello, world」。因為這本書影響極為深遠（C 語言是無數現代語言的祖先），這個範例就成了傳統，被後來幾乎所有語言的教學沿用。

（其實 Kernighan 更早在 1972 年的一份教材就用過類似範例，但 K&R 這本書讓它真正紅遍全球。）

## 為什麼是個好的「第一個程式」

「印出 Hello World」看似簡單到無聊，但它其實是個**絕佳的第一課**：

- **最小可行**：它是「能跑起來的最小程式」——讓你驗證「環境裝好了、能跑程式了」（呼應你各課 Part 0 的「跑出第一個程式」）。
- **沒有複雜邏輯**：不用懂變數、迴圈，專注在「怎麼寫、怎麼執行、怎麼看到輸出」這個最基本的循環。
- **即時回饋**：你寫了、跑了、馬上看到「Hello, World!」出現——那種「我讓電腦聽我的話了！」的成就感，是學程式的起點。

所以它不只是傳統，更是教學設計上的智慧——**先用最簡單的方式，讓你體驗「完整跑通一個程式」**，建立信心和環境，再往上疊複雜的東西。

## 它的文化意義

「Hello, World」成了程式設計的**文化符號**：

- 它象徵「**踏入程式世界的第一步**」——你寫出它的那一刻，就成了「會寫程式的人」。
- 學新語言時，大家還是習慣「先寫個 Hello World」找手感。
- 它甚至延伸出各種變體和玩笑（用最複雜的方式印 Hello World、用各種奇怪語言印…）。

下次你寫 `print("Hello, World!")`，可以想到——你正在參與一個從 1978 年延續至今、橫跨全球程式設計師的傳統。

## 小結

- 「Hello, World!」是全球程式設計師的「第一個程式」傳統。
- 來自 1978 年的經典著作《The C Programming Language》（K&R）的第一個範例。
- 它是絕佳的第一課：最小可行、無複雜邏輯、即時回饋、建立信心與環境。
- 它成了「踏入程式世界第一步」的文化符號。

> 你各課的 Part 0 都從「跑出第一個程式」開始——那就是你的 Hello World 時刻。
