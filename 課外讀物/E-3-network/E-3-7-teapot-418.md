# [E-3-7] 趣味：HTTP 狀態碼 418 — 我是茶壺

> **目標**：透過一個工程師的玩笑「418 我是茶壺」，輕鬆認識 HTTP 狀態碼，並感受工程文化的幽默。

## 一個正經系統裡的玩笑

你學過 HTTP 狀態碼（E-3-3、basic Part 4-B）——`200 OK`（成功）、`404 Not Found`（找不到）、`500 Internal Server Error`（伺服器錯誤）。它們都很正經。

但在這套嚴肅的標準裡，藏著一個**工程師的玩笑**——狀態碼 **`418 I'm a teapot`（我是茶壺）**。

## 故事的由來

這要回到 1998 年的愚人節。當時有人半開玩笑地寫了一份「正式」的網路協定文件（RFC 2324），叫做：

> **「超文字咖啡壺控制協定（HTCPCP，Hyper Text Coffee Pot Control Protocol）」**

這份文件「正經八百」地定義了「**如何用 HTTP 控制一個咖啡壺**」——例如用 `BREW`（沖煮）指令叫咖啡壺煮咖啡。而其中定義了一個狀態碼：

> **`418 I'm a teapot`**：當你「叫一個茶壺去煮咖啡」時，茶壺應該回傳這個狀態碼——意思是「**我是茶壺，不是咖啡機，我煮不了咖啡**」。

這純粹是個愚人節玩笑——「叫茶壺煮咖啡，它抗議說我是茶壺」。

## 它竟然「活」了下來

最妙的是——這個玩笑狀態碼**流傳了下來，還變成工程界的迷因（meme）**：

- 很多網路框架、函式庫真的**實作了 418**（你真的可以讓伺服器回傳「418 我是茶壺」）。
- 有些網站把它當彩蛋——某些網址會回傳 418 + 一個茶壺圖案。
- 2017 年，當有人提議「從標準移除 418」時，社群（包括很多工程師、甚至學生）強烈反對——「**Save 418！**」運動讓它被保留了下來。它成了工程文化的一部分。

## 它代表的工程文化

「418 茶壺」看似無聊，但它反映了工程界一個可愛的特質——**在嚴謹中保有幽默**：

- 工程師寫的東西大多嚴肅、精確（不能出錯）。
- 但他們也愛在裡面藏「彩蛋」「玩笑」「迷因」——left-pad 的釀酒命名（E-1-4 提到的 Homebrew 也是）、各種變數名 `foo`/`bar`（E-5-5）、Git 的故事（E-8-9）……
- 這些幽默讓嚴肅的工作多了人味，也成了工程師之間的「共同語言」和文化認同。

## 順帶複習：狀態碼的分類

藉這個玩笑，複習一下 HTTP 狀態碼的分類（E-3-3）——它們按「第一位數字」分類：

| 開頭 | 類別 | 例子 |
|------|------|------|
| **2xx** | 成功 | 200 OK |
| **3xx** | 重新導向 | 301 永久搬移、304 沒變（快取，cache-3-3）|
| **4xx** | **用戶端**錯誤 | 404 找不到、403 禁止、**418 我是茶壺** |
| **5xx** | **伺服器**錯誤 | 500 內部錯誤、503 服務不可用 |

418 被歸在 **4xx（用戶端錯誤）**——因為「叫茶壺煮咖啡」算是用戶端「搞錯對象」了。

## 小結

- `418 I'm a teapot` 源自 1998 愚人節的玩笑協定（用 HTTP 控制咖啡壺）。
- 意思：「叫茶壺煮咖啡 → 茶壺回『我是茶壺，煮不了』」。
- 它流傳成工程迷因，很多框架真的實作了它，還有「Save 418」運動保留它。
- 它反映工程文化「在嚴謹中保有幽默」的可愛特質。
