# [E-1-3] 檔案操作：cp / mv / cat / grep

> **目標**：學會終端機裡操作檔案的幾個核心指令——複製、移動、看內容、搜尋，這些是你每天都會用到的基本功。

## 四個天天用的指令

繼 E-1-2 的「移動」（cd/ls）之後，這篇是「操作檔案」的核心四招：

| 指令 | 全名 | 做什麼 |
|------|------|--------|
| `cp` | copy | 複製檔案/資料夾 |
| `mv` | move | 移動 / 改名 |
| `cat` | concatenate | 印出檔案內容 |
| `grep` | global regex print | 在檔案裡搜尋文字 |

## cp：複製

```bash
cp 原檔 目標            # 複製檔案
cp -r 原資料夾 目標     # 複製整個資料夾（-r = recursive 遞迴）
```

例：`cp config.txt config.bak`（備份一份）。複製資料夾一定要加 `-r`，否則它不知道要連裡面一起複製。

## mv：移動，也是改名

```bash
mv 檔案 目標資料夾/     # 移動
mv 舊名 新名            # 改名（移到「同一個地方但不同名字」= 改名）
```

有趣的是——**`mv` 同時是「移動」和「改名」**。因為「改名」本質就是「移到同一個位置、換個名字」。例：`mv old.txt new.txt` 就是改名。

> ⚠️ `cp` 和 `mv` 若目標已存在，會**直接覆蓋**、不問你。小心別蓋掉重要檔案。

## cat：看內容

```bash
cat file.txt           # 把整個檔案內容印到螢幕
```

`cat` 是 "concatenate（串接）"——它本來是「把多個檔案串起來印」，但最常用就是「快速看一個檔案的內容」。

> 檔案很大時，`cat` 會一次噴出全部、洗版。看大檔案用 `less file.txt`（可上下捲動，按 `q` 離開）或 `head`/`tail`（看頭/尾幾行）更好。

## grep：搜尋——最強大的一個

`grep` 在檔案裡「**找出含有某段文字的行**」，是工程師除錯、找東西的利器：

```bash
grep "error" log.txt           # 在 log.txt 找含 "error" 的行
grep -r "TODO" .               # 在當前資料夾「遞迴」找所有含 TODO 的（-r）
grep -i "error" log.txt        # 忽略大小寫（-i）
grep -n "error" log.txt        # 顯示行號（-n）
```

它最常配「管道 `|`」用——把別的指令輸出「倒給 grep 過濾」（你在 infra Part 2-3 用過）：

```bash
ps aux | grep nginx            # 從一堆程序中，只看含 nginx 的
```

`grep` 是除錯神器——日誌幾萬行，`grep "error"` 一下就找到出問題的那幾行。

## 小結

- `cp`（複製，資料夾要 `-r`）、`mv`（移動 / 改名）、`cat`（看內容）、`grep`（搜尋文字）。
- `cp`/`mv` 會直接覆蓋，小心。
- `grep` 最強大，常配管道 `|` 過濾其他指令的輸出，是除錯利器。
