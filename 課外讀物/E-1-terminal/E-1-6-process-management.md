# [E-1-6] 行程管理：ps / kill / 什麼是 port

> **目標**：理解「行程」是什麼，學會用 ps 看、用 kill 關，並搞懂「port（埠）被占用」是怎麼回事。

## 行程（Process）：正在跑的程式

**行程（process）** 就是「**正在執行的程式**」（infra Part 2-3 深入過，這裡是終端機角度的簡介）。你打開的每個程式、每個跑著的服務，都是一個行程，各有一個編號 **PID（Process ID）**。

## ps：看有哪些行程

```bash
ps aux                 # 列出所有行程（誰在跑、吃多少資源）
ps aux | grep node     # 配 grep（E-1-3）只看 node 相關的
```

輸出會顯示每個行程的 PID、CPU/記憶體用量、指令。最常用來「找出某個程式的 PID」或「看什麼在吃資源」。

> Mac 上也可以用內建的「活動監視器」App 看，但終端機的 `ps` / `top` 更快、能配指令用。

## kill：關掉行程

知道 PID 後，可以「關掉」一個行程：

```bash
kill 1234              # 禮貌地請 PID 1234 結束
kill -9 1234           # 強制砍掉（最後手段）
```

- `kill 1234`：送一個「請你結束」的訊號，程式可以收尾後退出。
- `kill -9 1234`：強制終止，不給收尾機會。**只在程式卡死、普通 kill 無效時才用**（呼應 infra Part 2-3）。

## Port（埠）：服務的「門牌號」

**Port（埠）** 是「一台機器上，識別『哪個服務』的號碼」（infra Part 3-1、basic Part 4 深入）。例如：

- 網頁伺服器常在 **port 3000、8080**。
- 資料庫常在 **5432**（PostgreSQL）。

一個 port「**同一時間只能被一個程式占用**」——這就帶出最常見的困擾。

## 「port 已被占用」怎麼辦

你跑一個服務，可能遇到：

```
Error: listen EADDRINUSE: address already in use :::3000
```

意思是「**port 3000 已經被別的程式占用了**」——通常是你「之前開的、忘了關」的同一個服務還在跑。解法：

```bash
# 找出「誰占用了 port 3000」
lsof -i :3000          # 列出用 port 3000 的行程（含 PID）

# 關掉它
kill <那個PID>
```

`lsof -i :3000`（list open files，看哪個行程開了這個 port）找出占用者的 PID，再 `kill` 掉，port 就釋放了，你的服務就能啟動。這是開發時超常用的排錯招式。

## 前景與背景

順帶一提，終端機跑程式有「前景」和「背景」：

```bash
node server.js         # 前景跑——占住終端機，Ctrl+C 可停
node server.js &       # 背景跑（加 &）——不占終端機，繼續用
```

前景程式按 `Ctrl+C` 就能停掉（送結束訊號）。背景程式要用 `kill` 或找 PID 處理。

## 小結

- 行程 = 正在跑的程式，每個有 PID。
- `ps aux`（配 grep）看行程；`kill PID` 關掉（`kill -9` 強制，最後手段）。
- **Port** = 服務的門牌號，同時只能一個程式占用。
- 「port 已被占用」→ `lsof -i :3000` 找占用者 → `kill` 它。
- 前景（占終端機，Ctrl+C 停）vs 背景（加 `&`）。
